## 2. 意图识别路由（IntentRouter）

### 2.1 设计目标

在用户输入进入 Agent Loop 之前，先进行意图识别。如果用户的输入可以被直接执行（内置命令、明确的工具调用），则直接执行并返回结果，不需要调用 LLM，从而降低延迟和成本。

### 2.2 三类意图

```
用户输入
  │
  ▼
┌─────────────────────────────────────────────┐
│            IntentRouter.route(input)          │
│                                               │
│  规则匹配（无 LLM 调用，< 1ms）               │
│  ┌─────────────────────────────────────┐     │
│  │ 1. 斜杠命令: /help, /model, /clear...│     │
│  │ 2. 明确工具调用: "读取 X 文件"        │     │
│  │ 3. 自然语言（兜底 → Agent Loop）      │     │
│  └─────────────────────────────────────┘     │
└─────────────────────────────────────────────┘
        │
        ├── 意图类型 1: 内置命令 → CommandHandler 直接执行
        ├── 意图类型 2: 直接工具 → ToolRegistry.execute() + 权限
        └── 意图类型 3: 自然语言 → Agent Loop (LLM)
```

### 2.3 路由规则定义

```typescript
// packages/core/src/router/intent-router.ts

/** 意图类型 */
enum IntentType {
    /** 内置斜杠命令: /help, /model, /clear... */
    BUILTIN_COMMAND = "builtin_command",
    /** 明确工具调用: "读取 X", "搜索 Y" */
    DIRECT_TOOL = "direct_tool",
    /** 自然语言，需要 LLM 推理 */
    NATURAL_LANGUAGE = "natural_language",
}

/** 路由结果 */
interface RouteResult {
    type: IntentType;
    /** 匹配到的处理器或工具 */
    handler?: CommandHandler | { toolName: string; params: Record<string, unknown> };
    /** 元数据：匹配规则、置信度等 */
    meta?: Record<string, unknown>;
}

class IntentRouter {
    private commandPatterns: Map<string, RegExp> = new Map();
    private toolPatterns: Map<string, { regex: RegExp; extractParams: (m: RegExpMatchArray) => Record<string, unknown> }> = new Map();

    constructor() {
        // ─── 注册内置命令 ───
        this.registerCommand("/help", /^\/help\b/);
        this.registerCommand("/clear", /^\/clear\b/);
        this.registerCommand("/model", /^\/model\b/);
        this.registerCommand("/config", /^\/config\b/);
        this.registerCommand("/memory", /^\/memory\b/);
        this.registerCommand("/skills", /^\/skills\b/);
        this.registerCommand("/review", /^\/review\b/);
        this.registerCommand("/init", /^\/init\b/);
        this.registerCommand("/fast", /^\/fast\b/);
        this.registerCommand("/exit", /^\/(exit|quit|q)\b/);

        // ─── 注册直接工具模式 ───
        // "读取/查看/打开 <文件路径>"
        this.registerToolPattern("read",
            /^(?:读取|查看|打开|看看|read|show|cat)\s+["']?(.+?)["']?\s*$/i,
            (m) => ({ file_path: m[1].trim() })
        );
        // "搜索/查找 <关键词> in <路径>"
        this.registerToolPattern("grep",
            /^(?:搜索|查找|grep|search)\s+["']?(.+?)["']?\s*(?:in\s+)?(.+)?$/i,
            (m) => ({ pattern: m[1].trim(), path: m[2]?.trim() || "." })
        );
        // "列出文件/显示目录"
        this.registerToolPattern("glob",
            /^(?:列出|显示|list|ls|dir)\s*(.+)?$/i,
            (m) => ({ pattern: m[1]?.trim() || "**/*" })
        );
        // "执行/运行 <命令>"
        this.registerToolPattern("bash",
            /^(?:执行|运行|run|exec)\s+(.+)$/i,
            (m) => ({ command: m[1].trim() })
        );
    }

    /** 路由用户输入，返回意图类型和处理方式 */
    route(input: string): RouteResult {
        // 1. 优先匹配内置命令
        for (const [cmd, pattern] of this.commandPatterns) {
            if (pattern.test(input)) {
                return { type: IntentType.BUILTIN_COMMAND, handler: this.getCommandHandler(cmd) };
            }
        }

        // 2. 匹配直接工具调用模式
        for (const [toolName, { regex, extractParams }] of this.toolPatterns) {
            const match = input.match(regex);
            if (match) {
                return {
                    type: IntentType.DIRECT_TOOL,
                    handler: { toolName, params: extractParams(match) },
                };
            }
        }

        // 3. 兜底：自然语言，进入 Agent Loop
        return { type: IntentType.NATURAL_LANGUAGE };
    }
}
```

### 2.4 路由决策流程

```typescript
// packages/core/src/router/dispatcher.ts

class RouterDispatcher {
    constructor(
        private router: IntentRouter,
        private agentLoop: AgentLoop,
        private toolRegistry: ToolRegistry,
        private permissionManager: PermissionManager,
    ) {}

    async dispatch(input: string, config: AgentConfig): Promise<DispatchResult> {
        const route = this.router.route(input);

        switch (route.type) {
            case IntentType.BUILTIN_COMMAND:
                // 直接执行命令，不走 LLM
                return this.executeCommand(route.handler as CommandHandler, input);

            case IntentType.DIRECT_TOOL: {
                // 直接调用工具 + 权限检查
                const { toolName, params } = route.handler as { toolName: string; params: Record<string, unknown> };
                const tool = this.toolRegistry.get(toolName);
                if (!tool) break; // 工具不存在，回退到 Agent Loop

                // 权限确认
                const approved = await this.permissionManager.check({ id: "0", name: toolName, input: params });
                if (!approved) {
                    return { type: "denied", message: "用户拒绝执行" };
                }

                // 直接执行工具
                const result = await tool.execute(params, this.buildContext(config));
                return { type: "tool_result", result };
            }

            case IntentType.NATURAL_LANGUAGE:
            default:
                // 进入 Agent Loop（调用 LLM）
                return { type: "agent_loop", stream: this.agentLoop.run(input, config) };
        }
    }
}
```

### 2.5 匹配策略

| 优先级 | 匹配方式 | 延迟 | 命中条件 |
| --- | --- | --- | --- |
| 1 | 前缀匹配 `/` | < 0.1ms | 以 `/` 开头，命中内置命令表 |
| 2 | 正则匹配中文/英文工具模式 | < 1ms | 匹配预定义的直接工具调用模式 |
| 3 | 兜底 LLM | ~1-3s | 以上均不命中，自然语言需要推理 |

## 3. Docker 沙箱设计（独立包 `@y-claude-code/sandbox`）

### 3.1 架构原则

core 包不依赖 dockerode，只依赖 `ISandbox` 接口。沙箱实现由 CLI 启动时创建并注入。

```
@y-claude-code/cli
    │
    ├── 创建 DockerSandboxManager (依赖 dockerode)
    │
    ▼
AgentConfig { sandbox: ISandbox }
    │
    ▼
@y-claude-code/core (只认识 ISandbox 接口)
    │
    ▼
BashTool.execute() → context.sandbox.exec(command, options)
```

### 3.3 DockerSandboxManager

```typescript
// packages/sandbox/src/manager.ts

import Docker from "dockerode";

class DockerSandboxManager implements ISandbox {
    private docker: Docker;
    private imageName: string;
    private imageReady: boolean = false;

    /** 安全策略 */
    private policy: SecurityPolicy;

    constructor(options: {
        imageName?: string;         // 默认 "node:20-alpine"
        socketPath?: string;        // 默认 /var/run/docker.sock
        policy?: Partial<SecurityPolicy>;
    }) {
        this.docker = new Docker({ socketPath: options.socketPath ?? "/var/run/docker.sock" });
        this.imageName = options.imageName ?? "node:20-alpine";
        this.policy = { ...DEFAULT_POLICY, ...options.policy };
    }

    /** 预热：拉取镜像 + 验证，在 CLI 启动时调用 */
    async warmup(): Promise<void> {
        const images = await this.docker.listImages();
        const exists = images.some(img => img.RepoTags?.includes(this.imageName));
        if (!exists) {
            // 流式拉取镜像，进度输出到日志
            await this.pullImage();
        }
        this.imageReady = true;
    }

    async exec(command: string, options: SandboxOptions): Promise<SandboxResult> {
        if (!this.imageReady) await this.warmup();

        // 1. 安全策略检查
        this.policy.validateCommand(command);
        this.policy.validateMounts(options.mounts ?? []);

        // 2. 创建容器
        const container = await this.docker.createContainer({
            Image: this.imageName,
            Cmd: ["sh", "-c", command],
            WorkingDir: options.workdir,
            Env: this.envToArray(options.env),
            HostConfig: {
                Binds: this.mountsToBinds(options.mounts),
                Memory: this.parseMemory(options.memoryLimit ?? this.policy.defaultMemoryLimit),
                NanoCpus: this.parseCpu(options.cpuLimit),
                NetworkMode: options.networkMode ?? this.policy.defaultNetworkMode,
                AutoRemove: true,  // 执行完自动清理
            },
        });

        const startTime = Date.now();

        try {
            // 3. 启动容器
            await container.start();

            // 4. 等待执行完成或超时
            const timeout = options.timeout ?? 120000;
            const result = await container.wait({ condition: "not-running" });

            // 5. 获取日志
            const logs = await container.logs({
                stdout: true, stderr: true,
                tail: 10000, // 最多 10000 行
            });

            const { stdout, stderr } = this.splitLogs(logs);
            const maxLen = options.maxOutput ?? 1_000_000;

            return {
                exitCode: result.StatusCode,
                stdout: stdout.slice(0, maxLen),
                stderr: stderr.slice(0, maxLen),
                timedOut: false,
                duration: Date.now() - startTime,
            };
        } catch (error) {
            if (error.message?.includes("timeout")) {
                await container.kill();
                return {
                    exitCode: -1,
                    stdout: "",
                    stderr: "命令执行超时",
                    timedOut: true,
                    duration: Date.now() - startTime,
                };
            }
            throw error;
        }
    }

    async isAvailable(): Promise<boolean> {
        try {
            await this.docker.ping();
            return true;
        } catch {
            return false;
        }
    }

    async getStatus(): Promise<{ image: string; running: boolean; version: string }> {
        const info = await this.docker.info();
        return {
            image: this.imageName,
            running: this.imageReady,
            version: info.ServerVersion,
        };
    }
}
```

```typescript
// packages/core/src/tools/builtin/bash.ts

class BashTool extends Tool {
    name = "Bash";
    description = "在 Docker 沙箱中执行 Shell 命令";

    async execute(params: Record<string, unknown>, context: ToolContext): Promise<ToolResult> {
        const command = params.command as string;
        const timeout = (params.timeout as number) ?? 120000;

        // 通过 ISandbox 接口调用，不感知 Docker 实现
        const result = await context.sandbox.exec(command, {
            workdir: context.workingDirectory,
            timeout,
            networkMode: "none", // Bash 默认无网络
            mounts: [{
                host: context.workingDirectory,
                container: "/workspace",
                mode: "rw",
            }],
        });

        if (result.timedOut) {
            return {
                tool_use_id: "",
                content: `命令超时（${timeout}ms），已被强制终止`,
                is_error: true,
            };
        }

        const output = [result.stdout, result.stderr ? `\n[stderr]\n${result.stderr}` : ""]
            .filter(Boolean).join("");

        return {
            tool_use_id: "",
            content: output || `(退出码: ${result.exitCode})`,
            is_error: result.exitCode !== 0,
        };
    }

    requiresApproval(): boolean { return true; }
}
```

### 4.1 SessionManager

```typescript
// packages/core/src/session/manager.ts

interface SessionData {
    id: string;
    createdAt: string;
    updatedAt: string;
    messages: Message[];
    tokenUsage: TokenUsage;
    workingDirectory: string;
    model: string;
    metadata: Record<string, unknown>;
}

class SessionManager {
    private currentSession: SessionData | null = null;
    private store: SessionStore; // 持久化后端

    constructor(store: SessionStore) {
        this.store = store;
    }

    /** 创建新会话 */
    create(workingDirectory: string, model: string): SessionData {
        this.currentSession = {
            id: crypto.randomUUID(),
            createdAt: new Date().toISOString(),
            updatedAt: new Date().toISOString(),
            messages: [],
            tokenUsage: { inputTokens: 0, outputTokens: 0 },
            workingDirectory,
            model,
            metadata: {},
        };
        return this.currentSession;
    }

    /** 恢复历史会话 */
    async resume(sessionId: string): Promise<SessionData> {
        const data = await this.store.load(sessionId);
        if (!data) throw new Error(`会话 ${sessionId} 不存在`);
        this.currentSession = data;
        return data;
    }

    /** 追加消息并自动持久化 */
    async appendMessage(message: Message): Promise<void> {
        if (!this.currentSession) throw new Error("无活跃会话");
        this.currentSession.messages.push(message);
        this.currentSession.updatedAt = new Date().toISOString();
        await this.store.save(this.currentSession);
    }

    /** 列出所有历史会话 */
    async list(): Promise<Array<{ id: string; createdAt: string; preview: string }>> {
        return this.store.list();
    }

    /** 删除会话 */
    async delete(sessionId: string): Promise<void> {
        await this.store.delete(sessionId);
    }

    /** 获取当前会话 */
    get current(): SessionData | null {
        return this.currentSession;
    }
}
```

### 4.2 存储后端

```typescript
// 文件系统存储实现
class FileSessionStore implements SessionStore {
    private baseDir: string;

    constructor(baseDir: string = "~/.y-claude-code/sessions") {
        this.baseDir = baseDir.replace("~", os.homedir());
    }

    async save(session: SessionData): Promise<void> {
        const path = `${this.baseDir}/${session.id}.json`;
        await fs.promises.writeFile(path, JSON.stringify(session, null, 2));
    }

    async load(sessionId: string): Promise<SessionData | null> {
        const path = `${this.baseDir}/${sessionId}.json`;
        try {
            const data = await fs.promises.readFile(path, "utf-8");
            return JSON.parse(data);
        } catch {
            return null;
        }
    }

    async list(): Promise<Array<{ id: string; createdAt: string; preview: string }>> {
        const files = await fs.promises.readdir(this.baseDir);
        const sessions = [];
        for (const file of files.filter(f => f.endsWith(".json"))) {
            const data = JSON.parse(await fs.promises.readFile(`${this.baseDir}/${file}`, "utf-8"));
            sessions.push({
                id: data.id,
                createdAt: data.createdAt,
                preview: this.extractPreview(data.messages),
            });
        }
        return sessions.sort((a, b) => b.createdAt.localeCompare(a.createdAt));
    }

    async delete(sessionId: string): Promise<void> {
        await fs.promises.unlink(`${this.baseDir}/${sessionId}.json`).catch(() => {});
    }

    private extractPreview(messages: Message[]): string {
        const firstUserMsg = messages.find(m => m.role === "user");
        if (!firstUserMsg) return "(空会话)";
        const content = typeof firstUserMsg.content === "string"
            ? firstUserMsg.content
            : firstUserMsg.content.map(b => ("text" in b ? b.text : "")).join("");
        return content.slice(0, 80);
    }
}
```

```typescript
// packages/core/src/agent/loop.ts

class AgentLoop {
    private state: AgentState = AgentState.IDLE;
    private messages: Message[] = [];
    private toolRounds: number = 0;

    /** 运行 Agent 主循环，返回异步事件流 */
    async *run(userInput: string, config: AgentConfig): AsyncGenerator<TurnEvent> {
        this.state = AgentState.IDLE;
        this.toolRounds = 0;

        // 1. 构建初始上下文
        this.messages = [
            { role: "system", content: config.systemPrompt },
            ...this.buildToolDefinitions(config.tools),
            { role: "user", content: userInput },
        ];

        while (this.toolRounds < config.maxToolRounds) {
            // 2. 调用 LLM（流式）
            this.state = AgentState.THINKING;
            const chunks = config.provider.chat(this.messages, {
                model: config.model,
                maxTokens: config.maxTokensPerTurn,
                tools: config.tools.map(t => this.toolToSchema(t)),
                thinking: config.thinkingEnabled,
                thinkingTokens: config.thinkingTokens,
            });

            let toolUses: ToolUse[] = [];
            for await (const chunk of chunks) {
                switch (chunk.type) {
                    case "text":
                        yield { type: "text", content: chunk.content };
                        break;
                    case "thinking":
                        yield { type: "thinking", content: chunk.content };
                        break;
                    case "tool_use":
                        toolUses.push({
                            id: chunk.id,
                            name: chunk.name,
                            input: chunk.input as Record<string, unknown>,
                        });
                        break;
                    case "error":
                        yield { type: "error", error: new Error(chunk.message) };
                        this.state = AgentState.ERROR;
                        return;
                }
            }

            // 3. 无工具调用 → 对话结束
            if (toolUses.length === 0) {
                this.state = AgentState.DONE;
                yield { type: "done", usage: this.getUsage() };
                return;
            }

            // 4. 执行工具调用（串行执行，保证顺序）
            const toolResults: ToolResult[] = [];
            for (const toolUse of toolUses) {
                yield { type: "tool_call", tool: toolUse };

                // 权限检查
                this.state = AgentState.WAITING_APPROVAL;
                const approved = await config.permissionManager.check(toolUse);
                if (!approved) {
                    toolResults.push({
                        tool_use_id: toolUse.id,
                        content: "用户拒绝了此操作",
                        is_error: true,
                    });
                    continue;
                }

                // 执行工具
                this.state = AgentState.EXECUTING;
                const tool = this.findTool(toolUse.name, config.tools);
                if (!tool) {
                    toolResults.push({
                        tool_use_id: toolUse.id,
                        content: `未知工具: ${toolUse.name}`,
                        is_error: true,
                    });
                    continue;
                }
                const result = await tool.execute(toolUse.input, this.buildContext(config));
                toolResults.push(result);
                yield { type: "tool_result", result };
            }

            // 5. 将工具结果追加到上下文
            this.messages.push({
                role: "assistant",
                content: toolUses.map(tu => ({
                    type: "tool_use" as const,
                    id: tu.id,
                    name: tu.name,
                    input: tu.input,
                })),
            });
            this.messages.push({
                role: "user",
                content: toolResults.map(tr => ({
                    type: "tool_result" as const,
                    tool_use_id: tr.tool_use_id,
                    content: tr.content,
                    is_error: tr.is_error,
                })),
            });

            this.toolRounds++;
        }

        // 达到最大轮次
        yield { type: "error", error: new Error(`达到最大工具调用轮次 (${config.maxToolRounds})`) };
        this.state = AgentState.DONE;
    }
}
```

### 5.3 Stopper 停止条件

```
条件1: LLM 返回的 chunk 流中不包含任何 tool_use → 自然结束
条件2: toolRounds >= maxToolRounds → 强制停止（默认 50 轮）
条件3: AbortSignal 被触发 → 用户中断
```

```typescript
// packages/core/src/llm/anthropic.ts

class AnthropicProvider implements LLMProvider {
    private client: Anthropic;

    constructor(apiKey: string, baseURL?: string) {
        this.client = new Anthropic({ apiKey, baseURL });
    }

    async *chat(messages: Message[], options: ChatOptions): AsyncIterable<ResponseChunk> {
        // system 消息单独提取（Anthropic API 要求）
        const systemMessages = messages.filter(m => m.role === "system");
        const conversationMessages = messages.filter(m => m.role !== "system");

        const systemPrompt = systemMessages
            .map(m => (typeof m.content === "string" ? m.content : m.content.map(b => b.text).join("")))
            .join("\n\n");

        // 构建请求
        const request: Anthropic.MessageCreateParams = {
            model: options.model,
            max_tokens: options.maxTokens,
            system: this.buildCacheableContent(systemPrompt),
            messages: this.convertMessages(conversationMessages),
            tools: options.tools?.map(this.convertTool),
            stream: true,
        };

        // Extended thinking 配置
        if (options.thinking && this.supportsFeature("thinking")) {
            request.thinking = {
                type: "enabled",
                budget_tokens: options.thinkingTokens ?? 4000,
            };
        }

        const stream = this.client.messages.stream(request);

        try {
            for await (const event of stream) {
                switch (event.type) {
                    case "content_block_delta":
                        if (event.delta.type === "text_delta") {
                            yield { type: "text", content: event.delta.text };
                        } else if (event.delta.type === "thinking_delta") {
                            yield { type: "thinking", content: event.delta.thinking };
                        } else if (event.delta.type === "input_json_delta") {
                            // 工具参数流式累积，由上层组装
                        }
                        break;
                    case "content_block_start":
                        if (event.content_block.type === "tool_use") {
                            // 记录 tool_use 开始，准备接收参数
                        }
                        break;
                    case "content_block_stop":
                        // 一个完整的 block 结束
                        break;
                    case "message_stop":
                        yield { type: "stop", reason: "end_turn" };
                        break;
                    case "error":
                        yield { type: "error", code: "anthropic_error", message: event.error.message };
                        break;
                }
            }
        } finally {
            stream.abort(); // 确保资源释放
        }
    }

    /** 构建可缓存的 system prompt（Anthropic Prompt Caching） */
    private buildCacheableContent(text: string): Array<TextBlock & { cache_control?: object }> {
        return [
            { type: "text", text, cache_control: { type: "ephemeral" } },
        ];
    }
}
```

```typescript
// packages/core/src/llm/openai.ts

class OpenAIProvider implements LLMProvider {
    private client: OpenAI;

    constructor(apiKey: string, baseURL?: string) {
        this.client = new OpenAI({ apiKey, baseURL });
    }

    async *chat(messages: Message[], options: ChatOptions): AsyncIterable<ResponseChunk> {
        const stream = await this.client.chat.completions.create({
            model: options.model,
            max_tokens: options.maxTokens,
            messages: this.convertMessages(messages),
            tools: options.tools?.map(this.convertTool),
            stream: true,
        });

        // 累积工具调用参数（OpenAI 的 tool_calls 是多次 delta 累积的）
        const pendingToolCalls = new Map<number, { id: string; name: string; arguments: string }>();

        for await (const chunk of stream) {
            const delta = chunk.choices[0]?.delta;

            if (delta?.content) {
                yield { type: "text", content: delta.content };
            }

            if (delta?.tool_calls) {
                for (const tc of delta.tool_calls) {
                    const index = tc.index;
                    const existing = pendingToolCalls.get(index) ?? { id: "", name: "", arguments: "" };

                    if (tc.id) existing.id = tc.id;
                    if (tc.function?.name) existing.name += tc.function.name;
                    if (tc.function?.arguments) existing.arguments += tc.function.arguments;

                    pendingToolCalls.set(index, existing);

                    // 当一个 tool_call 参数完整时，尝试解析并 yield
                    if (existing.id && existing.name && tc.function?.arguments?.endsWith?.("}")) {
                        try {
                            const input = JSON.parse(existing.arguments);
                            yield {
                                type: "tool_use",
                                id: existing.id,
                                name: existing.name,
                                input,
                            };
                            pendingToolCalls.delete(index);
                        } catch {
                            // 参数还不完整，继续累积
                        }
                    }
                }
            }

            if (chunk.choices[0]?.finish_reason === "stop") {
                yield { type: "stop", reason: "end_turn" };
            }
        }
    }
}
```

## 8. 上下文管理详细设计

### 8.1 Token 预算模型

```
总预算: contextWindow(model) — 例如 Claude Sonnet 4 = 200000 tokens

分配策略:
┌───────────────┬──────────┬─────────────────────────┐
│ 区域           │ 比例     │ 说明                     │
├───────────────┼──────────┼─────────────────────────┤
│ system_prompt │ ~5-10%   │ 固定，标记 cache_control │
│ tool_defs     │ ~3-5%    │ 工具定义，标记 cache     │
│ messages      │ ~70-75%  │ 对话历史                 │
│ output_reserve│ ~10-15%  │ 为 LLM 输出预留          │
└───────────────┴──────────┴─────────────────────────┘

当 messages 超出配额时，触发压缩：
  1. 保留最近 5 轮工具调用完整内容
  2. 对更早的轮次调用 Compressor 生成摘要
  3. 摘要格式："[之前，用户请求了 XX，助手通过 YY 工具完成了 ZZ]"
```

### 8.2 Summarizer 摘要策略

实际实现使用 `Summarizer`（文件: `packages/core/src/context/summarizer.ts`），采用 LLM 驱动的渐进式摘要，比以下设计更进一步（详见 01-系统架构 第 11 节）。以下为原设计的 Compressor 思路：

```typescript
// packages/core/src/context/compressor.ts (设计参考，实际见 summarizer.ts)

class ContextCompressor {
    /**
     * 压缩历史消息
     * 策略：使用一个轻量级 LLM 调用将旧消息压缩为摘要
     * 如果压缩后的 token 数仍然超标，则丢弃最早的轮次
     */
    async compress(
        messages: Message[],
        maxTokens: number,
        provider: LLMProvider,
    ): Promise<Message[]> {
        // 1. 计算当前总 token
        const totalTokens = await provider.countTokens(messages);

        if (totalTokens <= maxTokens) return messages;

        // 2. 找到分割点：保留最后 10 轮，前面压缩
        const recentCount = 20; // 最近 20 条消息（约 10 轮对话）
        const oldMessages = messages.slice(0, -recentCount);
        const recentMessages = messages.slice(-recentCount);

        // 3. 对旧消息生成摘要
        const oldSummary = oldMessages
            .filter(m => m.role !== "system")
            .map(m => {
                const content = typeof m.content === "string"
                    ? m.content
                    : m.content.map(b => ("text" in b ? b.text : "[工具数据]")).join("");
                return `[${m.role}]: ${content.substring(0, 200)}`;
            })
            .join("\n");

        const summaryPrompt = `请将以下对话历史压缩为一段简洁的摘要，保留关键信息：\n${oldSummary}`;

        // 使用 provider 生成摘要（需要一次 LLM 调用）
        const summaryResult = await this.generateSummary(summaryPrompt, provider);

        const compressedMessage: Message = {
            role: "user",
            content: `[对话历史摘要]: ${summaryResult}`,
        };

        // 4. 重组: system + 摘要 + 最近消息
        const systemMessages = messages.filter(m => m.role === "system");
        return [...systemMessages, compressedMessage, ...recentMessages];
    }
}
```

## 9. 权限系统详细设计

### 9.1 权限规则引擎

```typescript
// packages/core/src/permission/rules.ts

type PermissionLevel = "readonly" | "write" | "exec" | "network" | "all";

interface PermissionRule {
    /** 匹配的工具名称（支持 glob 模式） */
    toolPattern: string;
    /** 允许还是拒绝 */
    action: "allow" | "deny" | "ask";
    /** 作用域 */
    scope: "session" | "project" | "user" | "global";
    /** 可选的指令模式匹配 */
    commandPattern?: string;
}

class PermissionManager {
    private rules: PermissionRule[] = [];
    /** session 级别的决策缓存（单次 allow all 记忆） */
    private sessionCache: Map<string, "allow" | "deny"> = new Map();

    /** 检查工具调用是否需要用户确认 */
    async check(toolUse: ToolUse): Promise<boolean> {
        // 1. 查 session 缓存（用户勾选了"always allow in this session"）
        const cacheKey = this.buildCacheKey(toolUse);
        if (this.sessionCache.has(cacheKey)) {
            return this.sessionCache.get(cacheKey) === "allow";
        }

        // 2. 按优先级匹配规则：session > project > user > global
        for (const rule of this.sortedRules()) {
            if (this.matches(toolUse.name, rule.toolPattern)) {
                if (rule.action === "allow") return true;
                if (rule.action === "deny") return false;
                if (rule.action === "ask") {
                    // 触发交互式确认
                    return this.promptUser(toolUse);
                }
            }
        }

        // 3. 默认策略：需要确认
        return this.promptUser(toolUse);
    }

    /**
     * 记忆用户的本次选择
     * @param persistToScope 持久化级别 (session / project)
     */
    remember(toolUse: ToolUse, decision: "allow" | "deny", persistToScope?: "session" | "project"): void {
        if (persistToScope === "session") {
            this.sessionCache.set(this.buildCacheKey(toolUse), decision);
        }
        if (persistToScope === "project") {
            this.rules.push({
                toolPattern: toolUse.name,
                action: decision,
                scope: "project",
            });
            // 持久化到 .claude/settings.json
            this.persistRules();
        }
    }

    private buildCacheKey(toolUse: ToolUse): string {
        // Bash: 按命令基名做细粒度缓存（允许 npm 不会连带允许 rm）
        if (toolUse.name === "Bash" && typeof toolUse.input.command === "string") {
            const cmdBase = toolUse.input.command.split(" ")[0];
            return `Bash:${cmdBase}`;
        }
        // Write/Edit: 按文件路径细化（确认写入 fileA 不会连带放行 fileB）
        if ((toolUse.name === "Write" || toolUse.name === "Edit") && toolUse.input.file_path) {
            return `${toolUse.name}:${String(toolUse.input.file_path)}`;
        }
        return toolUse.name;
    }

    /** 以指定 key 写入 session 缓存（用于"本会话全允许"等宽泛记忆） */
    rememberWithKey(key: string, decision: "allow" | "deny"): void {
        this.sessionCache.set(key, decision);
    }

    /** 预检：判断工具调用是否会触发用户确认弹窗（同步、非阻塞） */
    willPromptUser(toolUse: ToolUse): boolean {
        const cacheKey = this.buildCacheKey(toolUse);
        if (this.sessionCache.has(cacheKey)) {
            return false; // 已有缓存决策，不会弹窗
        }
        // ... 规则匹配 ...
        return this.defaultMode === "ask";
    }
}
```

## 10. CLI 详细设计

### 10.1 技术选型

CLI 从 Ink (React TUI) 迁移到 Node.js 原生 `readline` 模块，原因：
- Ink 在部分终端环境下 `useInput` 不触发、中文输入一次只能输入一个字符
- Yoga 布局引擎在终端高度自适应场景下输入框定位不可靠
- readline 提供内建的终端历史、行编辑、提示符，零额外依赖

### 10.2 输入处理流程

```
用户输入 (stdin)
    │
    ▼
readline.on("line")
    │
    ▼
sanitizeInput() — 输入净化
    │
    ├── 以 "/" 开头 → executeCommand() → 内置命令处理 → 输出结果
    │
    └── 自然语言 → RouterDispatcher.dispatch()
                      │
                      ├── DIRECT_TOOL → 直接执行工具 → 输出结果
                      │
                      └── NATURAL_LANGUAGE → AgentLoop.run()
                                                │
                                                ├── 流式文本 → process.stdout.write()
                                                ├── 工具调用 → console.log("🔧 ...")
                                                ├── 权限确认 → 终端提示
                                                └── 错误 → console.log("❌ ...")
```

### 10.3 输出渲染规格

```
用户输入:
  > [用户输入文本]（紫色 ANSI 提示符）

Agent 流式回复:
  纯文本通过 StreamingMarkdownRenderer 渲染：
    — 代码块语法高亮（关键字/字符串/注释/数字 ANSI 着色）
    — **粗体** `行内代码` #标题 等行内格式转 ANSI
    — 连续空白行压缩：最多保留 2 个连续空行，超过丢弃（防止大片空白区域）

工具调用（带参数展示）:
  ⚙ 读取文件 src/app.ts 执行中...
  ⚙ 执行命令 npm test 执行中...
  ⚙ 搜索文件 **/*.ts 执行中...

权限确认（终端交互）:
  🔐 需要执行命令执行: npm install react，是否允许？
  (y=允许 / n=拒绝 / a=本会话全允许此类操作):
    — y → 允许本次，并记忆此具体操作（同文件/同命令基名），后续自动放行
    — a → 允许本会话中此工具的所有调用（如"全允许所有 Bash"）
    — n → 拒绝

批准/拒绝反馈:
  ✓ 已允许（后续相同操作自动放行）
  ✓ 已允许（本次会话自动放行所有 命令执行）
  ✗ 已拒绝

错误:
  [红色] 错误信息

行清除机制:
  所有事件输出前通过 writeLine() 清除当前行：
  readline.cursorTo(0) + readline.clearLine(0) + 写入文本 + \n
  避免裸 \r 导致的 spinner 字符残留。
```

### 10.4 快捷键映射

```
Ctrl+C          — 中断当前操作
Ctrl+D          — 退出程序
Ctrl+R          — 搜索历史命令
↑/↓             — 历史命令翻阅
Enter           — 发送消息
\ 结尾           — 多行输入续行
```

readline 的 `historySize: 1000` 配置自动提供历史持久化（内存级），终端原生的行编辑（左右移动、删除等）由 readline 内建支持。
