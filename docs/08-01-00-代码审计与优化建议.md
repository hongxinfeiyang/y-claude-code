### 🔴 **1. Agent Loop 中的规划闸门实现过于复杂且有副作用** `[已修复]`
**位置**：`packages/core/src/agent/loop.ts` (Line 221-263)

**问题**：
```typescript
private enforcePlanningGate(toolName: string): { blocked: boolean; message?: string }
```
- 依赖 `TodoWriteTool` 的**全局静态状态**（`hasActivePlan()`, `trackModifyCallWithoutPlan()` 等）
- 直接操作 `TodoWriteTool` 的静态方法会产生**副作用**，违反纯函数原则
- 规划状态应该属于 Agent 的会话级状态，不应该耦合在单个工具实例中

**建议**：
- 将规划状态（当前计划、修改计数、漂移检测）提升到 `AgentLoop` 或 `AgentLoopContext`
- 改为 `enforcePlanningGate(toolName: string, planState: PlanState)` 的纯函数形式
- 解耦 `TodoWriteTool` 和规划逻辑

### 🟡 **2. 工具结果截断策略过于粗糙** `[已修复]`
**位置**：`packages/core/src/agent/loop.ts` (Line 792-818)

**问题**：
```typescript
const estimatedTokens = Math.ceil(contentStr.length / CHARS_PER_TOKEN_ESTIMATE);
if (estimatedTokens <= MAX_TOOL_RESULT_TOKENS) return r;
const maxChars = MAX_TOOL_RESULT_TOKENS * CHARS_PER_TOKEN_ESTIMATE;
const truncated = contentStr.slice(0, maxChars);
```

- **Token 估算不准确**：英文 4 字符/token，中文通常是 1-2 字符/token，JSON 结构化数据比例不同
- **盲目截断**：可能在 JSON 对象中间截断，导致解析失败
- **信息丢失**：对于结构化输出（如 grep 结果），随意截断会失去关键信息

**建议**：
- 使用 `js-tiktoken` 进行**精确 token 计数**（已在 `package.json` 中）
- 对 JSON 结果按结构化方式截断（截断数组元素而非字符串中间）
- 对二进制/长列表结果采用**采样**而非简单截断

### 🟡 **3. 权限管理中的 Bash 命令细粒度检查有漏洞** `[已修复]`
**位置**：`packages/core/src/permission/manager.ts` (Line 392-401)

**问题**：
```typescript
private buildCacheKey(toolUse: ToolUse): string {
    if (toolUse.name === "Bash" && typeof toolUse.input.command === "string") {
        const cmdBase = toolUse.input.command.split(" ")[0];  // ← 只取第一个 word
        return `Bash:${cmdBase}`;
    }
```

- 当命令是 `npm ci && npm test` 时，只会提取 `npm`，无法区分不同的子命令风险
- `sudo npm` 会提取 `sudo` 而非 `npm`，权限绕过风险
- 复杂命令管道（如 `cat file | sed 's/x/y/' > output`）的意图检测缺失

**建议**：
- 使用命令行解析库（如 `yargs-parser`）正确解析命令和选项
- 对 `sudo` 等提权工具做特殊检测
- 支持管道和重定向的风险识别

### 🟡 **4. 配置加载器的 merge 逻辑只合并一层** `[已修复]`
**位置**：`packages/core/src/config/loader.ts` (Line 251-268)

**问题**：
```typescript
if (typeof srcVal === "object" && !Array.isArray(srcVal) && typeof tgtVal === "object" && !Array.isArray(tgtVal)) {
    // 嵌套对象：展开合并（保留 target 未覆盖的字段）
    const merged = { ...tgtVal, ...srcVal };  // ← 只合并一层
    (target[key] as Record<string, unknown>) = merged;
}
```

- **假设配置结构只有 2 层**，但当添加新的嵌套配置时容易出错
- `settings.json` 中的 `providers.anthropic.baseURL` 与 `providers.anthropic.apiKey` 会被覆盖而非合并
- 如果后续有 `providers.anthropic.customHeaders` 等新字段，会被意外覆盖

**建议**：
```typescript
// 实现深层合并或使用成熟库（lodash.merge）
function deepMerge(target: any, source: any): void {
    for (const key in source) {
        if (source[key] && typeof source[key] === 'object' && !Array.isArray(source[key])) {
            target[key] = target[key] || {};
            deepMerge(target[key], source[key]);
        } else {
            target[key] = source[key];
        }
    }
}
```

### 🟡 **6. Permission 规则匹配在 write/edit 工具中的歧义** `[已修复]`
**位置**：`packages/core/src/permission/manager.ts` (Line 398-399)

**问题**：

```typescript
if ((toolUse.name === "Write" || toolUse.name === "Edit") && toolUse.input.file_path) {
```

- **没有做路径规范化**：`./src/app.ts` 和 `src/app.ts` 会被当作不同的权限 key

```typescript
const normalizedPath = path.resolve(toolUse.input.file_path as string);
return `${toolUse.name}:${normalizedPath}`;
```

### 🟡 **7. 工具注册表中的重复注册检查不够严格** `[已修复]`
**位置**：`packages/core/src/tools/registry.ts` (Line 19-24)

**问题**：
```typescript
register(tool: Tool): void {
    if (this.tools.has(tool.name)) {
        throw new Error(`工具 "${tool.name}" 已注册，不允许重复注册`);
    }
    this.tools.set(tool.name, tool);
}
```

- **无法支持运行时热加载工具**：开发时编辑工具代码需要重启
- **没有提供替换/升级机制**：一旦注册无法修改
- 与现代"插件系统"的模式不符

**建议**：
```typescript
register(tool: Tool, options?: { replace?: boolean }): void {
    if (this.tools.has(tool.name) && !options?.replace) {
        throw new Error(`工具 "${tool.name}" 已注册`);
    }
    this.tools.set(tool.name, tool);
}
```

### 🟡 **8. ContextMonitor 告警阈值硬编码为 85%** `[已修复]`
**位置**：`packages/core/src/agent/loop.ts` (Line 751)

**问题**：
```typescript
if (status.usagePercent >= 85) {
    loopCtx.logger?.warn(
        `[AgentLoop] 上下文使用率 ${status.usagePercent.toFixed(1)}%，触发主动压缩`,
```

```typescript
const threshold = config.contextCompressThreshold ?? 0.85;
if (status.usagePercent >= threshold * 100) { ... }
```
