# 05 — 开发手册

## 1. 环境搭建

### 1.1 前置依赖

- Node.js >= 20.0.0（推荐使用 fnm / nvm 管理版本）
- pnpm >= 8.0.0（`npm install -g pnpm`）
- Git

### 1.2 克隆并初始化

```bash
git clone <repo-url>
cd y-claude-code

# 安装依赖
pnpm install

# 构建所有包
pnpm build

# 运行 CLI（开发模式）
pnpm --filter @y-claude-code/cli dev
```

### 1.3 开发环境配置

```bash
# 设置测试用 API Key
export ANTHROPIC_API_KEY="sk-ant-xxx"
export OPENAI_API_KEY="sk-xxx"

# 可选：设置本地 API 代理以降低测试成本
export ANTHROPIC_BASE_URL="http://localhost:8080"
```

## 3. 开发流程

### 3.1 添加新工具

```typescript
// packages/core/src/tools/builtin/my-tool.ts

import { Tool, ToolContext, ToolResult } from "../../types/tools.js";
import type { JSONSchema } from "../../types/tools.js";

export class MyTool extends Tool {
    // ─── 元数据 ───
    name = "my_tool";
    description = "工具的功能描述，LLM 据此判断何时调用";

    parameters: JSONSchema = {
        type: "object",
        properties: {
            param1: { type: "string", description: "参数1说明" },
            param2: { type: "number", description: "参数2说明", default: 10 },
        },
        required: ["param1"],
    };

    // ─── 执行逻辑 ───
    async execute(params: Record<string, unknown>, context: ToolContext): Promise<ToolResult> {
        const { param1, param2 } = params as { param1: string; param2?: number };

        // 实现逻辑...

        return {
            tool_use_id: "",  // 由上层填充
            content: `结果: ${param1}`,
            is_error: false,
        };
    }

    // ─── 是否需要用户确认 ───
    requiresApproval(params: Record<string, unknown>): boolean {
        // 返回 true 则执行前会弹出确认提示
        return false;
    }
}
```

然后在 `registry.ts` 中注册：

```typescript
import { MyTool } from "./builtin/my-tool.js";

static createDefault(): ToolRegistry {
    // ...
    registry.register(new MyTool());
    return registry;
}
```

## 4. 代码规范

### 4.1 TypeScript

- 严格模式 (`strict: true`)
- 禁止 `any` 类型（使用 `unknown` 替代）
- 公共 API 必须标注返回类型
- 接口命名不加 `I` 前缀
- 枚举使用 `const enum` 优先

### 4.2 注释规范

```typescript
/**
 * Agent 主循环 — 实现 ReAct 模式的推理-行动循环
 * @param userInput — 用户输入的原始文本
 * @param config — Agent 运行配置
 * @returns 异步事件流，按序产出文本/工具调用/错误/完成事件
 * @example
 * ```typescript
 * const loop = new AgentLoop();
 * for await (const event of loop.run("修复 bug", config)) {
 *     console.log(event);
 * }
 * ```
 */
async *run(userInput: string, config: AgentConfig): AsyncGenerator<TurnEvent> {
    // ─── 构建初始上下文 ───
    const messages = this.buildInitialMessages(userInput, config);
    // ...
}
```

### 4.3 命名约定

| 类别 | 规范 | 示例 |
|------|------|------|
| 文件名 | kebab-case 或 按模块目录分组 | `agent/loop.ts`, `intent-router.ts` |
| 类名 | PascalCase | `AgentLoop` |
| 函数/变量 | camelCase | `buildContext` |
| 常量 | UPPER_SNAKE_CASE | `DEFAULT_MAX_ROUNDS` |
| 接口 | PascalCase | `AgentConfig` |
| 类型别名 | PascalCase | `ResponseChunk` |

### 4.4 缩进

使用 4 个空格进行缩进（遵循 CLAUDE.md 约束）。
