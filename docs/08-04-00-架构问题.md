### **新问题 3: ToolExecutor 中的状态回调设计有问题**

**严重性**: 🟡 中等  
**位置**: `packages/core/src/agent/tool-executor.ts` (L69-70)

```typescript
async *executeAll(
    toolUses: ToolUse[],
    ctx: ToolExecutorContext,
    onStateChange: (state: string) => void,  // ❌ 状态用字符串表示
    onConsecutiveError: (isError: boolean) => void,

**问题**:

}

// 回调使用强类型
onStateChange: (state: ToolExecutorState) => void,

```

### **新问题 4: AgentServices 向后兼容的冗余检查**

**严重性**: 🟡 低-中等  
**位置**: `packages/core/src/agent/loop.ts` 各处  
**问题描述**:

为了向后兼容，代码中充满了这样的模式:

```typescript
// L315
const traceId = (loopCtx.services?.observability ?? loopCtx.observability)?.startTurn(userInput) ?? "";

// L323
(loopCtx.services?.observability ?? loopCtx.observability)?.endTurn(traceId, ...);

// L334
const alert = (loopCtx.services?.contextMonitor ?? loopCtx.contextMonitor)?.getAlert();

// L336
const status = (loopCtx.services?.contextMonitor ?? loopCtx.contextMonitor)!.getStatus();

// ... 重复出现 10+ 次
```

**问题**:
- 代码重复率高（DRY 原则违反）
- 每处都需要写 `(loopCtx.services?.xxx ?? loopCtx.xxx)`
- 难以阅读和维护

**建议修复**:
```typescript
// 在 run() 初始化时统一处理
private getService<T>(
    serviceName: 'observability' | 'contextMonitor' | 'errorRecovery' | ...
): T | undefined {
    return loopCtx.services?.[serviceName] ?? loopCtx[serviceName];
}

// 使用时
const observability = this.getService('observability');
observability?.startTurn(userInput);
```
