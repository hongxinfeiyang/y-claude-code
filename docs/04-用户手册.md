# 04 — 用户手册

## 1. 安装

### 1.1 环境要求

- Node.js >= 20.0.0
- npm >= 10.0.0 或 pnpm >= 8.0.0

### 1.2 安装方式

```bash
# npm 全局安装
npm install -g @y-claude-code/cli

# 或使用 pnpm
pnpm add -g @y-claude-code/cli

# 验证安装
y-claude-code --version
y-claude-code --help
```

### 1.3 初次启动 — 自助配置 LLM

首次启动时，y-claude-code 会引导你通过 UI 界面完成 LLM 配置，无需手动编辑配置文件：

```text
$ y-claude-code

欢迎使用 y-claude-code！检测到您是首次使用，请完成初始配置。

┌────────────────────────────────────────────┐
│          LLM Provider 配置                  │
│                                             │
│  选择 Provider:                             │
│  [1] Anthropic (Claude)                     │
│  [2] OpenAI (GPT)                           │
│  [3] 自定义兼容接口                          │
│                                             │
│  API Key: [________________]                │
│  Base URL (可选): [________________]        │
│  默认模型: [claude-sonnet-4-6________]      │
│                                             │
│  [ 测试连接 ]  [ 保存并继续 ]                │
└────────────────────────────────────────────┘
```

配置保存在 `~/.y-claude-code/config.json`，后续启动自动加载。你也可以随时通过以下方式修改：

- **UI 方式**: 运行 `y-claude-code --setup` 重新进入配置向导
- **命令方式**: `/model` 切换模型，`/config` 修改各项参数
- **文件方式**: 直接编辑 `~/.y-claude-code/config.json`
- **环境变量**: 设置 `ANTHROPIC_API_KEY` / `OPENAI_API_KEY` 覆盖配置

### 1.4 VS Code 插件安装

在 VS Code 扩展市场搜索 "y-claude-code" 并安装，或：

```bash
code --install-extension y-claude-code.vscode
```

## 4. 配置指南

### 4.1 配置文件位置

```
项目级: .claude/settings.json          # 对当前项目生效
项目级: .claude/settings.local.json    # 本地覆盖（不入 git）
用户级: ~/.y-claude-code/config.json           # 对所有项目生效
```

### 4.2 常用配置

```json
{
    "model": "claude-sonnet-4-6",
    "maxToolRounds": 50,
    "maxTokensPerTurn": 16000,
    "thinkingEnabled": false,
    "thinkingTokens": 4000,
    "permissions": {
        "defaultMode": "ask"
    },
    "theme": "dark",
    "env": {
        "NODE_ENV": "development"
    }
}
```

### 4.3 配置模型

```bash
# 查看当前模型
y-claude-code config get model

# 切换模型
y-claude-code config set model claude-opus-4-7

# 设置 API 端点（自定义代理）
y-claude-code config set anthropicBaseURL "https://your-proxy.com"
```

### 4.4 权限配置

```json
{
    "permissions": {
        "defaultMode": "ask",
        "rules": [
            { "toolPattern": "Read", "action": "allow", "scope": "project" },
            { "toolPattern": "Glob", "action": "allow", "scope": "project" },
            { "toolPattern": "Grep", "action": "allow", "scope": "project" },
            { "toolPattern": "Bash", "action": "ask", "scope": "project" }
        ]
    }
}
```

## 5. 工具使用说明

### 5.1 让 Agent 读取文件

```
> 看看 src/app.ts 里面有什么
> 读取 package.json 的依赖
```

Agent 自动使用 Read 工具。

### 5.2 让 Agent 修改代码

```
> 把 handleSubmit 函数重命名为 handleFormSubmit
> 给所有 API 调用添加错误处理
```

Agent 使用 Edit 工具进行精确替换。

### 5.3 让 Agent 执行命令

```
> 运行测试看有没有失败的
> 安装 lodash 包
> 查看 git 提交历史
```

Agent 使用 Bash 工具执行命令，需要你确认。

**权限确认快捷键**:
| 按键 | 行为 | 有效期 |
|------|------|--------|
| `y` | 允许本次，并记住此具体操作（同命令基名/同文件路径），后续自动放行 | 当前 session |
| `n` | 拒绝本次执行 | — |
| `a` | 本会话全允许此类操作（如所有 Bash 或所有 Write），不再弹窗 | 当前 session |

> 示例：允许 `npm test` → 后续 `npm install`、`npm run build` 等所有 npm 命令自动放行（`Bash:npm` 缓存 key）。但 `rm -rf` 仍需要确认（`Bash:rm` 是独立的缓存 key）。

### 5.4 让 Agent 搜索代码

```
> 找到所有使用 useState 的地方
> 搜索 TODO 注释
```

Agent 使用 Grep 工具。

## 6. 高级功能

### 6.1 Plan Mode（计划模式）

Plan Mode 让你在执行复杂任务前先审核 AI 的实现计划：

1. Agent 判断任务复杂度较高时，自动进入计划模式
2. 在计划模式下，Agent 只能使用只读工具（Read/Glob/Grep）调研代码
3. 调研完成后，Agent 通过 ExitPlanMode 提交实现计划
4. 你审核计划后选择批准或拒绝
5. 批准后 Agent 按计划逐步实现

你也可以在对话中直接要求："请先制定计划，不要直接修改代码"。

### 6.2 使用 Skill

Skill 是预定义的专业能力模块，启动时自动加载。Agent 根据任务自动判断何时应用相应 Skill：

**内置 Skill 示例**: `code-review`（代码审查）、`test-generator`（测试生成）、`refactor`（代码重构）、`debug`（调试）、`docs`（文档生成）

**查看可用 Skill**: `/skills`

**自定义 Skill**：在 `.claude/skills/` 目录下创建 Markdown 文件：

```markdown
---
name: my-deploy
description: 自定义部署流程
level: project
---

当用户请求部署时：
1. 运行 npm run build
2. 运行 npm test
3. 执行 npm run deploy
4. 汇报部署结果
```

加载优先级：用户级 (`~/.y-claude-code/skills/`) > 项目级 (`.claude/skills/`) > 内置

### 6.3 上下文管理

长对话中系统自动进行上下文压缩，保留关键信息：

- **自动压缩**: 当上下文使用率达到阈值（默认 70%）时自动触发 LLM 驱动的渐进式摘要
- **手动压缩**: 使用 `/compact` 命令随时触发
- **查看状态**: 使用 `/context` 查看当前上下文使用情况

### 6.4 Hook 自定义

在 `.claude/settings.json` 中配置 Hook：

```json
{
    "hooks": {
        "before:tool:execute": [
            {
                "matcher": "Bash",
                "command": "echo '审计: ${TOOL_PARAMS}' >> audit.log"
            }
        ]
    }
}
```

### 6.5 MCP 集成

配置外部工具服务器：

```json
{
    "mcpServers": {
        "filesystem": {
            "command": "npx",
            "args": ["-y", "@anthropic-ai/mcp-server-filesystem", "/path/to/data"]
        },
        "github": {
            "command": "npx",
            "args": ["-y", "@anthropic-ai/mcp-server-github"]
        }
    }
}
```

### 6.6 tmux 集成

如果在 tmux 环境中运行，y-claude-code 自动检测并支持：

- `/tmux info` — 查看 tmux 环境信息（版本、会话、窗口、面板）
- `/tmux windows` — 列出当前会话所有窗口
- `/tmux sessions` — 列出所有活跃 tmux 会话
- `/tmux capture` — 捕获当前面板内容供 Agent 分析

### 6.7 自动更新与遥测

- **自动更新检查**: 启动时异步检查 npm registry，有新版本时提示。可通过 `autoUpdateCheck: false` 关闭
- **遥测（匿名）**: 收集使用统计数据帮助改进产品。可通过 `telemetryEnabled: false` 关闭

## 7. 常见问题

### Q: 遇到问题如何诊断？
```bash
# 运行内置诊断工具
/doctor

# 或通过命令行查看配置
y-claude-code --setup

# 检查日志
tail -f ~/.y-claude-code/logs/*.log
```

### Q: API Key 如何设置？
```bash
export ANTHROPIC_API_KEY="sk-ant-xxx"
# 或在 ~/.y-claude-code/config.json 中设置
```

### Q: 如何切换 OpenAI？
```bash
y-claude-code config set provider openai
y-claude-code config set model gpt-4o
```

### Q: 如何限制费用？
```bash
y-claude-code config set maxTokensPerTurn 8000
y-claude-code config set maxToolRounds 20
```

### Q: 对话内容存储在哪里？
`~/.y-claude-code/sessions/` 目录下，每个会话一个 JSON 文件。

### Q: 如何恢复中断的会话？
```bash
y-claude-code --resume <session-id>
```

### Q: 如何更新？
```bash
npm update -g @y-claude-code/cli
```
