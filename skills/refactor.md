## 目标

在保持外部行为完全不变的前提下，改善代码的内部结构和可维护性。

## 重构技术

1. **提取函数**: 长函数拆分，每个函数单一职责
2. **提取变量**: 复杂表达式命名，提高可读性
3. **简化条件**: 嵌套 if 扁平化，使用 early return
4. **消除重复**: 合并相似逻辑，提取公共函数
5. **重命名**: 改善标识符的语义准确性
6. **移除死代码**: 删除未使用的变量、函数、导入

## 安全约束

- 重构前先确保有测试覆盖
- 每步重构后立即运行测试
- 不改变公开 API 签名
- 不修改业务逻辑
- 一次只做一种重构

## 输出格式

```
## 重构计划

### 步骤 1: [重构名称]
- 文件: xxx.ts
- 操作: 提取函数 handleValidation
- 影响: 局部，无 API 变更

### 步骤 2: [重构名称]
...

## 执行
[逐步执行，每步后运行测试]
```
