## 目标

从用户提供的错误信息出发，系统地定位问题根因并给出修复方案。

## 调试流程

1. **信息收集**
   - 读取完整的错误堆栈
   - 查看相关日志
   - 了解复现步骤

2. **假设建立**
   - 根据错误类型列出可能原因（最多 3 个假设）
   - 按可能性排序

3. **假设验证**
   - 读取相关源码验证每个假设
   - 必要时插入诊断日志或断点
   - 确认或排除假设

4. **根因确认**
   - 明确导致错误的精确代码行和逻辑
   - 解释为什么之前没有暴露

5. **修复方案**
   - 提供具体修复代码
   - 说明修复影响范围
   - 建议添加的测试

## 常见错误模式

| 错误 | 常见原因 | 优先检查 |
|------|----------|----------|
| `TypeError: Cannot read properties of undefined` | 缺少空值检查、异步数据未就绪 | 调用链上游 |
| `SyntaxError` | 不匹配的括号、引号 | 最近的代码变更 |
| 内存溢出 | 无限循环、大数组、闭包泄漏 | 循环条件、引用持有 |
| API 超时 | 网络问题、服务端慢查询 | 超时配置、重试逻辑 |

## 输出格式

```
## 错误分析

### 现象
[错误信息/堆栈]

### 假设
1. [假设1] — 可能性：高/中/低
2. [假设2] — 可能性：高/中/低

### 验证
[检查过程和结论]

### 根因
[文件名:行号] [具体原因]

### 修复
[修复方案和代码]

### 预防
[建议添加的测试/检查]
```
