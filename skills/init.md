## 目标

分析当前项目的代码结构、技术栈和架构模式，生成一份高质量的 CLAUDE.md 项目文档。

## 执行流程

1. **项目扫描**: 列出顶层目录结构，识别项目类型（monorepo/单体/库）
2. **技术识别**: 读取 package.json、tsconfig.json、配置文件，确定技术栈
3. **目录分析**: 识别源码目录、测试目录、文档目录、配置目录
4. **依赖分析**: 列出核心依赖，理解项目用途
5. **编码规范提取**: 从 eslint、prettier、editorconfig 等提取代码风格
6. **文档生成**: 按模板生成 CLAUDE.md

## 输出模板

```markdown
# 项目概述
# 技术栈
# 目录结构
# 编码规范
# 核心模块说明
# 构建与运行
# 注意事项
```
