name: test-generator

level: 模块级
---

## 目标

分析目标代码（函数/类/模块），生成高质量测试覆盖。

## 执行流程

1. **代码分析**: 读取目标文件，理解函数签名、分支逻辑、依赖关系
2. **场景识别**: 列出正向用例、边界条件、错误路径
3. **Mock 规划**: 确定需要 Mock 的外部依赖（API、文件系统、数据库）
4. **测试生成**: 使用 vitest/Jest 语法生成测试代码
5. **运行验证**: 执行生成的测试，确认通过

## 测试生成规则

- 每个公开函数至少 1 个正向用例 + 2 个边界用例
- 分支较多的函数，每个分支至少 1 个用例
- 异步函数需覆盖 resolve 和 reject
- Mock 外部依赖，不 Mock 被测对象本身
- 测试描述使用中文

## 输出格式

```typescript
// __tests__/xxx.test.ts
import { describe, it, expect, vi } from "vitest";

describe("TargetFunction", () => {
    // ─── 正向用例 ───
    it("正常输入应返回预期结果", () => { ... });

    // ─── 边界条件 ───
    it("空数组输入应返回 0", () => { ... });
    it("负数输入应抛出异常", () => { ... });

    // ─── 异常路径 ───
    it("外部依赖失败时应优雅降级", () => { ... });
});
```
